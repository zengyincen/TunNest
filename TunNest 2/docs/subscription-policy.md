# 囤囤（TunNest）订阅限制原则

这套限制以“用户知道自己为什么被限制，并且不会丢失既有内容”为核心：

1. 新用户无需绑卡，自动获得连续 7 天完整功能，不做数量阉割。
2. 扩展持续显示剩余试用天数，到期后才停止新的 Notion 写入。
3. 付费墙不遮挡设置和既有同步记录；已经写入 Notion 的页面永不删除、不加水印。
4. 所有套餐功能完全一致，只按时长定价，默认包含 3 个浏览器设备和 1 个独立 GitHub Actions 槽位。
5. 浏览器使用随机安装码，不采集硬件序列号、邮箱或 Chrome 账号信息；Worker 只保存哈希。
6. 用户可以在设置里主动释放本机槽位；无法操作旧设备时，由优先客服清空设备绑定。
7. 临时断网提供 24 小时宽限，但宽限不会延长已经到期的试用。
8. GitHub Actions 属于持续无人值守服务，不提供试用，验证失败时在读取平台数据和写入 Notion前立即退出。

建议运营政策：正常换机免费重置；明显共享同一许可证时先提醒再暂停；续费前 7 天提醒但不频繁弹窗。这样的限制比硬件指纹和突然锁死更容易获得长期用户信任。
